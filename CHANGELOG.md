# CHANGELOG

本项目遵循 `VMAJOR.MINOR.PATCH` 版本号。当前版本为 `V0.0.2`。

## V0.0.2 - 2026-07-14

### Added

- 新增基于 SQLite 的统一记忆数据库 `memory_store.py`。
- 新增消息持久化：保存收发方向、群 ID、群名称、成员、消息段、原始事件、trace ID 与事件时间。
- 新增群成员档案：自动聚合昵称、群名片、首次/最近出现时间、消息数量与最近发言。
- 新增成员人格备注：支持为不同群成员保存角色、语气、偏好、事实与自定义备注。
- 新增群级记忆：支持群摘要、长期事实、群规则和结构化记忆保存。
- 新增对话记忆文件：AI 服务重启后可恢复最近群聊上下文。
- 新增记忆中心统计：展示消息、群、成员、媒体、语音包及人格数据规模。
- 新增全文消息检索，可按群、成员、方向、时间和关键词过滤。
- 新增轻量向量检索，为历史消息生成本地确定性向量并按相似度召回。
- 新增记忆重建、JSON 导入和 JSON 导出能力。
- 新增图片、文件、视频、语音和表情消息的统一媒体索引。
- 新增异步媒体分析队列，媒体分析不会阻塞普通文本回复。
- 新增媒体元数据、状态、标签、关键词、OCR 文本、摘要及错误信息管理。
- 新增图片图库页面，支持群筛选、关键词搜索、预览、OCR、重新分析和人工标注。
- 新增语音内容页面，集中查看群语音、转写文本、状态、发送者与时间。
- 新增图片 OCR / Vision 配置、连通性测试、自动分析和手动重新识别。
- 新增 ASR 配置、连通性测试、自动转写和单条语音重新转写。
- 新增 SILK 到 WAV 转换工具源码，并在需要时自动构建本地转换器。
- 新增语音原始数据提取与下载缓存解密兜底，支持缺少回调音频时恢复语音文件。
- 新增可选的微信 UI 语音转文字观察器及启动、停止、构建脚本。
- 新增语音转写结果回写媒体记录，并可将转写文本按普通群消息进入 AI 回复链路。
- 新增语音包数据库、语音包分类、包来源、使用次数和最近使用时间记录。
- 新增从目录、子目录、`.zip`、`.zip1` 批量扫描和导入语音包。
- 新增语音标题清理、重复文件跳过、导入计划预览和导入结果汇总。
- 新增语音包搜索、分类筛选、推荐、音频预览、虚拟长列表和目标群发送。
- 新增 AI 语音包意图识别，兼容 `/发语音`、`[语音包]`、`[语音]` 等命令格式。
- 新增语音包查询清洗和相关性评分，可根据问候、道歉、情绪等意图匹配内容。
- 新增模型选择语音回复模式：发送语音成功后不再额外发送重复文字。
- 新增表情包/图片素材库，支持检索、OCR、标签、人工编辑和发送到授权群。
- 新增 AI 显式表情包请求处理，仅在用户明确请求时发送，避免随机刷屏。
- 新增统一消息发送中心，支持 `text`、`at`、`reply`、`image`、`file`、`video`、`record`。
- 新增本地路径、`file://`、HTTP URL 和 `base64://` 媒体输入归一化。
- 新增远程媒体下载的协议、大小、超时和内容长度限制。
- 新增 OneBot 完整链路测试：回调、探针、恢复、重启第二微信、@、引用、图片、文件、视频和语音。
- 新增链路 trace ID，并关联消息接收、AI 生成、渠道切换、媒体处理和 OneBot 发送日志。
- 新增最近链路视图，展示接收时间、生成时间、发送延迟、结果和监控状态。
- 新增群别名管理和从第二微信可见群聊同步群名称。
- 新增 OneBot 目标微信 PID 校验，确保 Hook 始终附加当前第二微信。
- 新增 OneBot `/health`、`/status` 和 `/media_probe` 本地诊断接口。
- 新增 OneBot 运行监控状态、媒体通道状态和 Web 后台手动恢复入口。
- 新增 UploadMedia 上下文的按微信 PID 缓存和状态展示。
- 新增微信 `4.1.11.53` 媒体服务定位地址配置。
- 新增 UploadMedia 上下文无界面自修复：通过微信内部 `default` 服务定位器解析当前进程媒体服务对象。
- 新增媒体服务未就绪时每秒自动重试，成功后立即写入当前 PID 缓存。
- 新增 OneBot 媒体上传回调状态、成功次数、失败次数、冷却时间和最后错误记录。
- 新增 AI 内置工具权限配置，可控制状态、日志、群、模型测试、探针、记忆检索、人格、媒体、语音包和表情包工具。
- 新增 AI 收发消息持久化、最近错误缓存和媒体分析工作线程。
- 新增 AI 文字发送重试与发送结果持久化。
- 新增 Web 后台记忆数据库、图片图库、语音内容、语音包管理和表情包管理页面。
- 新增 Web 后台 OCR、ASR、媒体自修复、OneBot 监控和链路追踪控制面板。
- 新增桌面与移动端媒体卡片、预览弹窗、内容展开、虚拟列表和响应式样式。

### Changed

- 扩展 AI 配置模型，加入 `memory`、`tools`、`vision_ocr`、`asr`、`onebot_monitor` 和 `media_auto_repair`。
- 扩展示例环境变量，加入独立 Vision OCR 与 ASR API Key。
- 扩展 OneBot 消息解析：无文本媒体消息也会被索引，文本与媒体可并行处理。
- 扩展 AI 上下文，加入长期记忆、成员人格、相关历史、图片 OCR、语音转写和可用语音包。
- 扩展模型渠道日志，所有渠道成功、失败、冷却和自动切换事件都携带 trace ID。
- 调整 OneBot 启动脚本，自动发现 `ffmpeg` 或 `imageio-ffmpeg` 并注入媒体转换 PATH。
- 调整 OneBot 启动脚本，默认关闭会干扰 Frida 会话的循环媒体探针；媒体上下文改由内部服务定位器恢复。
- 调整 OneBot 停止脚本，同时停止可选语音转文字 sidecar。
- 调整媒体发送超时，图片、视频和语音使用更符合真实上传耗时的等待时间。
- 调整媒体发送响应，返回传输方式、大小、摘要和延迟，同时对大段 base64 内容脱敏。
- 调整群聊发现逻辑，优先使用群别名并合并日志、记忆数据库和当前配置中的群。
- 调整 Web 管理后台导航和信息密度，新增媒体与记忆工作区并优化移动端布局。
- 将 OneBot 新增源码从本地构建目录同步回受版本控制的 vendor 快照，确保公开源码可复现构建。

### Fixed

- 修复 OneBot 重启后 Frida 会话进入 `context cancelled` 导致不再上报群消息的问题。
- 修复后台监控在 OneBot 尚未附加完成时反复重启进程的问题排查与启动时序。
- 修复只保存群 ID、不保存真实群名称导致后台难以辨认群聊的问题。
- 修复媒体消息没有文本时被 AI 入口直接丢弃、无法进入图库与记忆库的问题。
- 修复语音回调没有内嵌音频时无法保存和转写的问题。
- 修复 SILK 音频转换依赖系统 `ffmpeg`、缺失时直接失败的问题。
- 修复语音包导入只检查有限目录、无法识别大型嵌套包和 `.zip1` 的问题。
- 修复语音包搜索只查看最近记录、导致大语音库匹配遗漏的问题。
- 修复模型输出语音标记后仍把标记文字发到群里的问题。
- 修复语音发送完成后又额外发送重复文字回复的问题。
- 修复图片、视频和语音发送虽然成功但后台因超时显示失败的问题。
- 修复媒体上传上下文未初始化时只返回错误、无法进入恢复流程的问题。
- 修复从历史日志跨微信进程恢复 UploadMedia 指针造成 `access violation` 的问题。
- 修复旧 UploadMedia 指针 `0xc76decf98` 在新微信进程中被误判为可用的问题。
- 修复语音包上传返回 `upload voice failed` 时不会触发自动恢复与重试的问题。
- 修复语音发送失败时将内部 HTTP 400 和 OneBot JSON 原样发进群的问题。
- 修复 OneBot 媒体状态只依据日志判断、无法区分当前微信 PID 与旧缓存的问题。
- 修复 OneBot 下载媒体尾块不足 AES block size 时解密失败的问题。
- 修复部分 protobuf 语音数据包含前置字节时无法定位 `#!SILK_V3` 的问题。
- 修复群消息发送缺少重试、一次短暂网络错误就直接丢失的问题。
- 修复媒体图库中大段内容撑开卡片、长文本难以阅读的问题。

### Security

- UploadMedia 指针缓存必须同时匹配当前微信 PID 与合法指针格式，旧进程缓存会被拒绝。
- 删除无法证明进程身份的 `macos.log` 指针恢复兜底。
- 媒体下载仅允许 HTTP/HTTPS，限制重定向后的协议、文件大小、读取大小与超时。
- API 和页面响应会折叠 `base64://` 媒体正文，避免日志及浏览器泄露大量内容。
- `.gitignore` 新增真实配置快照、OneBot 备份、媒体缓存、编译后二进制和本地 Go 工具链规则。
- 将 Web 后台默认语音包路径改为公开占位符，移除本机用户名和目录。
- 将测试事件中的真实微信 self ID 改为 `wxid_test_bot` 示例值。
- 继续默认排除真实 API Key、真实 AI 配置、微信日志、语音包缓存和 OneBot 二进制。

### Verified

- 验证第二微信与 OneBot 附加 PID 一致，主微信未被附加。
- 验证 OneBot 与 AI 健康接口正常，进程监控期间 PID 保持稳定。
- 验证服务定位器可在全新微信进程启动后自动解析 UploadMedia 上下文并按 PID 缓存。
- 验证语音包 SILK 上传返回 `result=0`、微信语音发送返回 `result=1`、HTTP 返回 200。
- 验证文本群消息可正常接收、调用模型并通过 OneBot 发回目标群。
- 验证用户 crontab 中不存在每 30 分钟重启 OneBot 的任务。

## V0.0.1 - 2026-07-12

### Added

- 初始化 `wechat-mac-hook` 项目。
- 增加 macOS 第二微信隔离 Hook：
  - 拦截 `NSHomeDirectory()`、`NSSearchPathForDirectoriesInDomains(...)`、`getenv("HOME")`。
  - 拦截常见 C 文件 API，重定向第二微信相关路径。
  - 增加环境变量保护，仅在第二实例环境中生效。
- 增加第二微信安装、启动、状态检测脚本。
- 增加微信 macOS `4.1.11.53` OneBot 运行脚本与地址配置。
- 增加 OneBot -> AI -> OneBot 群聊回复桥接服务。
- 增加 Web 管理后台：
  - 运行总览。
  - 模型配置。
  - 群聊策略。
  - 测试中心。
  - 完整实时日志。
- 增加多模型渠道能力：
  - 渠道新增、保存、删除。
  - 模型列表获取。
  - 手动测试与自动健康检测。
  - 失败冷却与自动切换。
- 增加群权限白名单：只有勾选群才会触发 AI 回复。
- 增加机器人性格编辑器，并注入系统提示词。
- 增加示例配置：
  - `config/ai_reply_config.example.json`
  - `config/ai_reply.env.example`
- 增加项目规则文档 `RULES.md`。

### Changed

- 将 README 重写为公开仓库说明文档。
- 将桌面端 Swift 管理器的项目根目录改为环境变量 / 当前目录 / 源码路径自动推断，移除本机绝对路径。
- 将测试脚本默认群 ID 改为示例值，避免提交真实群信息。
