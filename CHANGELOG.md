# CHANGELOG

本项目遵循 `VMAJOR.MINOR.PATCH` 版本号。当前版本为 `V0.0.4`。

## V0.0.4 - 2026-07-15

本版本重点完成拍一拍自动回复的完整产品链路和微信原生动态表情快速发送，同时加入第二微信自动登录守护、强制触发阈值修正、账号身份自校正及更严格的 OneBot 收发健康判定。

### 拍一拍自动回复

- 新增拍一拍独立回复入口；事件命中后绕过接话评分、Reranker、回复任务审计和大模型生成，直接进入随机文字或随机图片回复。
- 新增后台拍一拍设置：总开关、随机文字开关、每行一条的多文本编辑、随机图片开关、同群最短间隔和保存状态提示，配置保存后热加载生效。
- 新增从表情素材库选择多张图片作为候选，支持滚动查看全部已选素材、点击移除及数量统计，不再只能管理前四张。
- 新增 PNG、JPEG、GIF、WebP 多文件上传；上传后自动导入表情资产、生成稳定文件摘要并立即加入拍一拍候选。
- 新增 `0` 秒同群最短间隔；设置为 `0` 时连续拍一拍逐次处理，不再静默丢弃第二次事件。
- 新增标准 OneBot `notice/poke` 与微信 `<sysmsg type="pat">` 两种事件格式兼容。
- 新增严格目标识别：读取 `<pattedusername>` 判断被拍成员，只有目标等于当前机器人 `self_id` 才回复；拍其他群友或目标不明确时一律旁路。
- 新增真实发起人识别：从 `<fromusername>` 提取拍一拍成员，避免把 `@chatroom` 群 ID 误当成人物 ID。
- 新增素材可靠性筛选，失效或连续失败素材不会优先进入即时随机池；一次拍一拍最多尝试一个素材，避免连续上传导致卡顿或微信异常。

### 微信原生动态表情发送原理

- 新增微信原生 `type=8` 表情消息构造器：使用表情稳定 MD5、原始文件长度和目标群构造 `sendappmsg` 请求，不再把收藏 GIF 当作普通图片重复上传。
- 原生表情由微信按照 MD5 从本地收藏/持久化表情层读取，因此不重新编码 GIF，不主动静态化动图，也不依赖图片上传产生的 CDN Key 与 AES Key。
- 新增 OneBot `/send_native_emoji`、`/native_emoji/status` 和 `/native_emoji/bind` 接口，返回原生通道就绪状态、账号、耗时和明确错误。
- 新增原生 `sendemoji` 捕获兼容层，用于诊断和兼容已学习模板；默认直接构造 `type=8` 消息，OneBot 重启后不需要用户重新发送表情校准。
- 原生表情绕过普通消息固定发送间隔、UploadMedia 单通道信号量和媒体上传预检，语音或图片上传卡顿不会阻塞拍一拍回复。
- 修复原生表情已被微信同步接受后仍等待普通图片 `buf2resp` 回调、最终误报 5 秒超时的问题；原生 StartTask 接受成功后立即返回。
- 拍一拍快速路径发送失败时不再降级到 15–20 秒的 UploadMedia 上传，避免重复发送、GIF 静态化和阻塞后续事件。
- 本机真实值班群端到端验证中，事件识别、素材选择和原生发送总耗时约 `43ms`；单独后台发送接口约 `50ms`。

### 第二微信自动登录

- 新增只针对 `~/Applications/WeChat2.app` 进程的自动登录守护，主微信不在检测和操作范围内。
- 新增基于本地 macOS Vision OCR 的登录窗识别，支持紧凑版“进入微信/登录”窗口并返回窗口与按钮的归一化坐标。
- 仅当窗口内容稀疏、按钮文本精确匹配且连续多次检测一致时才执行点击；普通聊天中出现“登录”文字不会触发。
- 新增检测间隔、点击冷却、连续确认次数和单轮最大尝试次数，点击后再次 OCR 确认登录按钮已经消失。
- 新增运行总览自动登录状态、脉冲指示、最近检测、错误说明、立即只读检测和启停开关。
- 新增 `/api/auto-login/status`、`/api/auto-login/config` 和 `/api/auto-login/check` 管理接口。

### OneBot 与回复链路改进

- OneBot 启动时从永久消息库选择最近一条真实 `self_id`，不再按历史出现次数选择已经退出的旧账号。
- 无法取得合法发送账号时拒绝启动媒体链路，避免使用错误 wxid 构造消息导致发送失败或微信异常。
- 收到新事件且 `self_id` 变化时运行时自动更新发送账号，支持第二微信切换账号后的自校正。
- 启动成功判定同时要求文本发送设置和接收回调均已完成，接收通道未就绪时不再误报 OneBot 正常。
- 媒体就绪状态同时校验 StartTask 与 UploadMedia 上下文，修复只有上传指针却无法完成发送时显示“已就绪”的问题。
- 停止健康监控自动排队文件传输助手媒体探针，避免一次卡住的上传占满 OneBot 串行 worker 并阻塞文字和原生表情。
- 修复喊“小风”等机器人别名已经被识别为强制触发，但最终仍被普通接话门槛拦截的问题；明确喊机器人和 @ 机器人继续不受评分阈值约束。
- 普通群消息仍严格执行最终评分硬门槛，不恢复随机概率或已删除的扣分规则。

### 测试与验证

- 新增 10 项拍一拍测试，覆盖标准通知、真实微信 XML、拍机器人、拍其他成员、未知目标、真实发起人、连续事件、文字随机、图片快速路径、素材失败和上传导入。
- 新增微信原生表情 protobuf 测试，验证 `type=8`、目标群、MD5、文件长度和非法素材拒绝。
- 新增第二微信自动登录测试，覆盖已登录页面保护、精确按钮识别、繁忙窗口拒绝和安全配置下限。
- 新增别名强制触发与普通消息阈值回归测试。
- 完成 Go 全量测试、Python 单元测试、Python/JavaScript 语法检查和运行时健康检查。
- 使用真实“成员拍其他成员”XML 验证 AI 返回不排队，OneBot 不产生任何表情发送任务。

## V0.0.3 - 2026-07-14

本版本将群聊回复链路升级为可并行、可追踪、具备永久记忆与自动媒介选择的“群聊大脑”，并补齐独立用户画像、语音包管理、表情索引、成员黑名单和后台可视化能力。

### 群昵称与界面细节修复（2026-07-15 补充）

- 修复群回复可能直接显示 `@wxid_xxx` 或误把 `@chatroom` 当作成员的问题；只有取得经过校验的真实昵称时才发送艾特。
- 新增艾特消息段的 `user_id` 与 `name` 字段，AI 将本地解析出的群昵称显式传递给 OneBot，不再依赖重启后为空的运行时昵称缓存。
- 无法确认真实昵称时继续正常回复正文但跳过艾特，禁止在群内泄露内部账号 ID、群 ID 或“未知成员”等占位名称。
- 修复部分旧微信用户名不以 `wxid_` 开头时被错误识别为群 ID 的问题，改为从群消息原始发送者前缀恢复真实成员 ID。
- 新增当前群优先的昵称解析：依次使用群名片、当前群显示名、昵称和引用消息证据，最后才允许使用跨群可读名称兜底。
- 新增引用消息 `<chatusr>` 与 `<displayname>` 配对回溯，可从全部永久历史中恢复并持久化成员真实显示名。
- 新增群成员目录合并写入方法，目录同步不会虚增成员消息次数，完整目录数据可实时补充昵称、群名片和显示名。
- 增加内部 ID、占位昵称、旧用户名、缓存昵称和引用昵称的自动化测试，确保 HTTP 与 WebSocket 发送路径规则一致。
- 深化明亮主题的统一蓝灰视觉层级，修复导航、Hero、卡片、终端、关系图、素材区和记忆指标仍残留深色或对比失衡的问题。
- 修复实时回复问题摘要、线程 ID 和用户画像成员信息过长时挤压、遮挡或溢出的问题，并补充完整内容悬停提示。
- 优化移动端群路由与回复摘要网格，在窄屏下保持边框、信息密度和触控区域一致。
